Отличная идея! Это решит главную проблему — FFmpeg WASM в браузере медленный (даже с ultrafast он едва дотягивает до real-time). На сервере FFmpeg будет работать в десятки раз быстрее.

Давайте спланируем архитектуру:

## Предлагаемая архитектура

### 1. Клиент (браузер) — подготовка данных
Вместо того чтобы рендерить кадры и кодировать видео на клиенте, клиент будет:
- Собирать **метаданные** проекта: клипы, обрезки, эффекты, аннотации, оверлеи, freeze-сегменты
- Отправлять их на сервер в виде JSON + исходные видеофайлы (или ссылки на них в S3)

### 2. Сервер — рендеринг и кодирование
Сервер (Next.js API route или отдельный сервис) будет:
- Получать метаданные проекта
- Скачивать исходные видео (с S3 или из загруженных файлов)
- Использовать **нативный FFmpeg** (не WASM) для рендеринга + кодирования
- Загружать готовый файл в S3
- Возвращать клиенту URL готового видео

### 3. Клиент — получение результата
- Клиент получает URL и показывает пользователю

## Варианты реализации

### Вариант A: Next.js API Route + FFmpeg на сервере
- Создать API route `/api/export` в `apps/demo`
- Сервер запускает FFmpeg через `child_process` или `fluent-ffmpeg`
- FFmpeg на сервере может использовать аппаратное ускорение (VideoToolbox на macOS, NVENC на GPU)
- **Плюс:** просто, в рамках существующего проекта
- **Минус:** долгий запрос (нужна поддержка streaming response или WebSocket для прогресса)

### Вариант B: Отдельный микросервис
- Node.js/Express сервис с FFmpeg
- Очередь задач (Bull/BullMQ + Redis)
- WebSocket для прогресса
- **Плюс:** масштабируется, не блокирует Next.js
- **Минус:** сложнее инфраструктура

### Вариант C: AWS Lambda + FFmpeg Layer
- Lambda с FFmpeg бинарём (через Lambda Layer)
- Trigger по S3 upload
- **Плюс:** serverless, платим только за использование
- **Минус:** лимит 15 минут на выполнение, меньше памяти/CPU

## Что нужно сделать в коде

1. **Клиент:** модифицировать `useExport.ts` — вместо `runExport()` отправлять POST на сервер с метаданными
2. **Сервер:** создать API route, который принимает метаданные, запускает FFmpeg, возвращает результат
3. **S3:** добавить загрузку исходных видео на S3 (сейчас они загружаются в браузер через FileReader)
